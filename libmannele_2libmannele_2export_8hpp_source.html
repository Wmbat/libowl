13 // inline functions, and classes with only inline member functions. Exporting
14 // classes that inherit from non-exported/imported bases (e.g., std::string)
15 // will end up badly. The only known workarounds are to not inherit or to not
16 // export. Also, MinGW GCC doesn't like seeing non-exported functions being
17 // used before their inline definition. The workaround is to reorder code. In
18 // the end it's all trial and error.
